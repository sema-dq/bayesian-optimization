Theorie der Bayesschen Optimierung

Kompakte Übersicht der wichtigsten Konzepte

Das Problem

Wir suchen das Minimum einer teuren Black-Box-Funktion f(x):

min f(x), wobei jede Auswertung von f(x) kostspielig ist

Kein Gradient verfügbar — nur Input → Output. Budget: typisch 10–50 Evaluierungen.

Gaussian Process (GP)

Das Surrogatmodell approximiert die unbekannte Funktion. Für jeden Punkt x liefert er:

μ(x) — Schätzung (Mittelwert)   |   σ(x) — Unsicherheit (Std.-Abweichung)

Der Kernel bestimmt die Glätte-Annahme. Matérn (ν=2.5) ist Standard in der Praxis. Der Length Scale steuert, wie weit der Einfluss eines Datenpunkts reicht.

Acquisition Functions

Entscheidet wo als nächstes gemessen wird. Balanciert Exploration vs. Exploitation:

Expected Improvement (EI)

E[max(f* − f(x), 0)]

Beliebteste Wahl, gute Balance. Parameter ξ steuert Exploration.

Upper Confidence Bound (UCB)

μ(x) − κ · σ(x)

Direkter Trade-off über Parameter κ. Höheres κ = mehr Exploration.

Probability of Improvement (PI)

P(f(x) < f*)

Einfach aber oft zu gierig — ignoriert die Größe der Verbesserung.

Der Algorithmus

1. Initialisiere mit n₀ zufälligen Samples → D₀ 2. Für t = 1, 2, ..., T: a) Fitte GP auf D_{t-1} b) Berechne Acquisition Function α(x) c) x_t = argmax α(x) ← Nächster Messpunkt d) y_t = f(x_t) + ε ← Teure Evaluation! e) D_t = D_{t-1} ∪ {(x_t, y_t)} 3. Gib x* = argmin y_i zurück

Anwendungen

Machine Learning: Hyperparameter-Tuning (jedes Training dauert Stunden)
Materialforschung: Optimale Legierung finden (jedes Experiment ist teuer)
Pharma: Wirkstoff-Dosierung (klinische Tests sind limitiert)
Robotik: Steuerungsparameter optimieren (physische Tests kosten Zeit)
Industrie: Fertigungsprozesse optimieren (Produktionsstopps vermeiden)
